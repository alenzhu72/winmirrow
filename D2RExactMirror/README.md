# D2R Exact Mirror

这是一个重新写的独立 WinForms 工具，用两个 D2R 窗口的人物屏幕位置做方向跟随。

## 热键

- `F11`：绑定当前前台窗口为窗口1。
- `F12`：绑定当前前台窗口为窗口2。
- `Ctrl+Shift+E`：启动/停止镜像。
- `Ctrl+Shift+Z`：停止镜像。
- `Ctrl+Shift+Q`：强制结束程序，先释放窗口2里的 `E`，再退出。
- `Ctrl+Shift+1`：记录窗口1人物点。
- `Ctrl+Shift+2`：记录窗口2人物点。
- `Ctrl+Shift+S`：交换带领者和跟随者。交换后 `E` 会发给新的跟随者。
- `Ctrl+Shift+A`：窗口2停止/恢复跟随。停止时窗口2释放 `E`，适合放在安全位置吃经验。
- `Shift`：带领窗口站住时，跟随窗口同步释放 `E` 并按住 `Shift`；松开后自动恢复按住 `E`。

如果热键被占用，直接用界面按钮：

- `自动绑定D2R窗口`：枚举标题包含 `Diablo II: Resurrected` 的可见窗口，屏幕左边的作为窗口1，右边的作为窗口2。
- `记录窗口1人物点`：把鼠标放到窗口1人物脚下/角色中心，点击按钮记录。
- `记录窗口2人物点`：把鼠标放到窗口2人物脚下/角色中心，点击按钮记录。
- `坐标:...`：切换坐标模式。`LegacyAbsolute` 是旧项目逻辑，把窗口1鼠标客户区坐标按比例映射到窗口2；`DirectionAnchor` 才使用人物点方向。
- `人物点:...`：切换人物点模式。仅 `DirectionAnchor` 需要。默认是 `Manual`，第一次运行需要先标记窗口1和窗口2人物位置；`DynamicPlayCenter` 每次按窗口尺寸自动取画面中间偏上位置。
- `鼠标:...`：切换鼠标发送方式，依次为 `PostMessage`、`SendMessage`、`FocusMessage`、`FocusCursorPulse`。
- `切换带领`：交换窗口1和窗口2。当前窗口1是带领者，当前窗口2是跟随者和 `E` 的目标。
- `窗口2:跟随/停止`：切换窗口2是否自动按住 `E` 跟随。
- `启动` / `停止`：同一个切换按钮，停止时显示启动，运行时显示停止。
- `强制停止`：只负责停止镜像并释放窗口2里的 `E`，用作兜底。

## 行为

- 使用两个窗口的真实客户区尺寸。
- `LegacyAbsolute` 模式下，鼠标在窗口1客户区 `(x,y)` 时，程序按两个窗口客户区比例换算成窗口2的 `(x,y)`。
- `DirectionAnchor` 模式下，程序先计算鼠标相对窗口1人物点的方向偏移，再用 `窗口2人物点 + 方向偏移 * 窗口尺寸比例`。
- 这样窗口2人物会朝窗口1人物鼠标方向移动，而不是死盯窗口同一个绝对坐标。
- 运行时会显示方向箭头：窗口1是绿色细箭头，表示输入方向；窗口2是蓝色细箭头，表示实际发送给窗口2的人物移动方向。
- 不把窗口外坐标夹到边缘，鼠标不在窗口1客户区内就不发送。
- 鼠标必须落在窗口1实际可见区域上；如果窗口2或工具窗口盖住了该位置，就不发送，避免重叠区域误判。
- 启动后只在窗口1有有效鼠标方向时让窗口2按住 `E`；窗口1不会被发送 `E`。
- 如果窗口1不是焦点、鼠标不在窗口1可见区域、或坐标无效，程序会立刻释放窗口2的 `E`，避免窗口2沿旧方向一直跑。
- 只有窗口1是焦点窗口时才发送鼠标移动。

## 人物点是什么

人物点就是每个窗口里角色脚下或角色身体中心的屏幕坐标，用它当移动方向的原点。比如窗口1鼠标在人物右上方，程序会把“右上方”这个方向发到窗口2人物的右上方，而不是发窗口里的同一个绝对位置。

默认使用 `Manual`。第一次运行需要先标记两边人物位置，否则不会启动发送。`DynamicPlayCenter` 适合 D2R 角色大多数时候保持在画面中间偏上的情况，可以手动切换尝试。

推荐用热键记录：

1. 鼠标放到窗口1角色脚下，按 `Ctrl+Shift+1`。
2. 鼠标放到窗口2角色脚下，按 `Ctrl+Shift+2`。

## 发送方式

- `PostMessage`：默认后台消息，最稳定但游戏可能不理。
- `SendMessage`：同步窗口消息，有些窗口比 `PostMessage` 更容易吃到。
- `FocusMessage`：短暂把焦点切到窗口2发送 `WM_MOUSEMOVE`，再切回之前窗口；不移动真实鼠标。
- `FocusCursorPulse`：短暂把焦点切到窗口2，把真实鼠标移动到蓝箭头目标点，按一下 `E`，再恢复鼠标和焦点。它最符合“按下 E，人物走向鼠标位置”的游戏输入逻辑，但会有轻微跳动。

`PostMessage`/`SendMessage` 不移动真实鼠标。`FocusMessage` 会有轻微焦点闪动。`FocusCursorPulse` 会短暂移动真实鼠标，仅在其他方式方向不对时尝试。
